# Virtual Goldberg 300 – Стандартные программы стерилизации

Версия: v0.1

Этот документ описывает набор типовых программ стерилизации для виртуального GOLDBERG 300.  
Реальная конфигурация может отличаться и должна уточняться по документации производителя и требованиям клиники.

## Общие поля программы

- `id` — строковый идентификатор (используется в коде).
- `name` — человекочитаемое название.
- `setTempC` — заданная температура стерилизации, °C.
- `sterilizationTimeSec` — время стерилизации при заданной температуре, в секундах.
- `preVacuumCount` — количество предвакуумов.
- `dryingTimeSec` — время сушки, в секундах.

## Примеры программ

### 1. Инструменты 134°C / 5 мин

```json
{
  "id": "prog_134_5",
  "name": "Инструменты 134°C / 5 мин",
  "setTempC": 134,
  "sterilizationTimeSec": 5 * 60,
  "preVacuumCount": 3,
  "dryingTimeSec": 10 * 60
}
```

Назначение: стандартная программа для стерилизации хирургических инструментов.

---

### 2. Текстиль 121°C / 20 мин

```json
{
  "id": "prog_121_20_textile",
  "name": "Текстиль 121°C / 20 мин",
  "setTempC": 121,
  "sterilizationTimeSec": 20 * 60,
  "preVacuumCount": 4,
  "dryingTimeSec": 15 * 60
}
```

Назначение: стерилизация текстильных материалов и упаковок.

---

### 3. Быстрый цикл 134°C

```json
{
  "id": "prog_134_fast",
  "name": "Быстрый цикл 134°C",
  "setTempC": 134,
  "sterilizationTimeSec": 3 * 60,
  "preVacuumCount": 3,
  "dryingTimeSec": 5 * 60
}
```

Назначение: быстрый цикл для срочных инструментов (при соблюдении требований).

---

### 4. Деликатная 121°C

```json
{
  "id": "prog_121_delicate",
  "name": "Деликатная 121°C",
  "setTempC": 121,
  "sterilizationTimeSec": 15 * 60,
  "preVacuumCount": 2,
  "dryingTimeSec": 5 * 60
}
```

Назначение: для термочувствительных материалов, допускающих 121°C, но не 134°C.

---

### 5. Тест Bowie–Dick

```json
{
  "id": "prog_bowie_dick",
  "name": "Тест Bowie–Dick",
  "setTempC": 134,
  "sterilizationTimeSec": 3 * 60,
  "preVacuumCount": 3,
  "dryingTimeSec": 0
}
```

Назначение: проверка эффективности удаления воздуха и проникновения пара.

---

### 6. Вакуум-тест

Для вакуум-теста обычно используется отдельный режим (не программа стерилизации в классическом смысле), но для удобства можно описать конфиг:

```json
{
  "id": "prog_vacuum_test",
  "name": "Вакуум-тест",
  "setTempC": 0,
  "sterilizationTimeSec": 0,
  "preVacuumCount": 1,
  "dryingTimeSec": 0
}
```

Дополнительные параметры вакуум-теста (время стабилизации, время теста, допустимая скорость утечки) задаются отдельно в конфигурации ядра или UI.

---

## Использование в коде

Программы могут храниться:

- либо как статический массив в `core-sterilizer`,
- либо как конфиг в JSON/YAML,
- либо в базе данных.

Минимальный интерфейс в коде:

```ts
interface ProgramConfig {
  id: string;
  name: string;
  setTempC: number;
  sterilizationTimeSec: number;
  preVacuumCount: number;
  dryingTimeSec: number;
}
```

UI и ядро используют `id` для ссылки на конкретную программу.
